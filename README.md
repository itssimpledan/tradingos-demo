# TradingOS — Options Trading System (Public Demo)

An end-to-end options trading workflow tool: screening, AI-assisted trade
recommendations, strategy backtesting, position tracking, and portfolio
analytics — built with Flask, SQLite, and the Anthropic API.

**This is a public demo deployment.** It runs on fictional sample data
(portfolio, trades, net worth) and synthetic market data, and is not
connected to any real account, brokerage, or live market feed. The AI
assistant is rate-limited per visitor to control API costs.

## Features

- **Dashboard** — portfolio value, P&L, allocation, and earnings calendar at a glance.
- **Screener** — theme-based equity screening with scoring.
- **Analyse** — per-ticker deep dive: valuation, momentum, predictability/moat scoring, multi-year charts.
- **Options Strategy** — AI-assisted strategy recommendations with payoff diagrams.
- **Backtest** — historical simulation of options strategies (percentage or fixed-dollar strike widths).
- **Tracker** — equities, crypto, cash, and options positions across multiple platforms.
- **Net Worth** — asset/liability tracking with monthly snapshots.
- **AI Assistant** — Claude-powered Q&A over your portfolio context.

## Tech stack

Flask · SQLite · pandas / numpy / scipy · yfinance · Anthropic Claude API · vanilla JS frontend.

## Running locally

```bash
pip install -r requirements.txt
python seed_demo_data.py   # populates options.db with sample data
python app.py              # http://localhost:5000
```

Set `ANTHROPIC_API_KEY` in your environment to enable the AI assistant.
Set `DEMO_MODE=1` to show the demo banner, cap the assistant, and use
synthetic market data instead of live yfinance calls (useful on hosts
that restrict outbound network access).

## Notes on this demo build

- All portfolio, trade, and net-worth data is fictional, generated by `seed_demo_data.py`.
- Market data is either live via yfinance, or — when `DEMO_MODE=1` on a
  network-restricted host — synthetic but realistic data generated by
  `demo_market_data.py`, so the app stays fully interactive.
- This is a separate deployment from the author's personal instance,
  which runs locally with real data and is never exposed publicly.
